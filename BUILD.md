# Building JDig locally (Android)

The app is an Expo **managed** project. The `android/` folder is generated by
`expo prebuild` and is gitignored, so a few release-build fixes must be
(re-)applied after prebuild. They are documented here for reproducibility.

## Prerequisites
- Node 20+, JDK 17 (or 21), Android SDK with **platform 34**, **build-tools 34.0.0**,
  **NDK 26.1.10909125**, **cmake 3.22.1** (install missing pieces via `sdkmanager`).
- `ANDROID_HOME` pointing at the SDK.

## First-time setup
```bash
npm install
npx expo prebuild --platform android        # generates android/
npm run build:editor                        # builds cm-bundle.js + editor.html into android/app/src/main/assets/
```

## Required release-build fixes (re-apply after each `expo prebuild`)

### 1. ProGuard keep rules — `android/app/proguard-rules.pro`
R8 otherwise strips RN's `fabric.Binding` and Expo modules → white screen on launch.
Append:
```proguard
-keep class com.facebook.react.** { *; }
-keep interface com.facebook.react.** { *; }
-keep class expo.modules.** { *; }
-keep class * implements com.facebook.react.bridge.NativeModule { *; }
-keep class * implements com.facebook.react.bridge.JavaModule { *; }
-keep class * implements com.facebook.react.turbo.NativeModule { *; }
-keepclassmembers class * { native <methods>; }
-keep class com.swmansion.reanimated.** { *; }
# react-native-webview bridge (R8 strips it in release → blank editor console/handshake)
-keep class com.reactnativecommunity.webview.** { *; }
-dontwarn com.reactnativecommunity.webview.**
-keep class androidx.webkit.** { *; }
-dontwarn androidx.webkit.**
```

### 2. `android/gradle.properties`
```properties
android.enableProguardInReleaseBuilds=true
android.enableShrinkResourcesInReleaseBuilds=true
reactNativeArchitectures=arm64-v8a,x86_64     # add x86,armeabi-v7a for universal
```

### 3. Patch `node_modules/expo-modules-core/android/ExpoModulesCorePlugin.gradle`
Change `from components.release` → `from components.findByName("release")`
(Expo SDK 52 ships no ProGuard consumer rules; this line otherwise breaks Gradle
config on Gradle 8.10). Consider `patch-package` to make it durable.

## Build the APK
```bash
npm run build:editor          # regenerate editor assets after any cm-source.js / build-editor.mjs change
cd android && ./gradlew assembleRelease
# -> app/build/outputs/apk/release/app-release.apk  (signed with debug.keystore)
```

## Install / run
```bash
adb install -r android/app/build/outputs/apk/release/app-release.apk
# dev iteration: npx expo start  (with a development build on the device)
```

## Notes
- The **CodeMirror editor** loads from packaged assets (`cm-bundle.js` +
  `editor.html` under `android/app/src/main/assets/`) via
  `source={{ uri: 'file:///android_asset/editor.html' }}` in
  `components/editor/editorHtml.ts`. This avoids react-native-webview's
  inline-`html` size truncation and inline-`<script>` HTML corruption — both of
  which blanked the editor in release. **Always run `npm run build:editor` before
  `assembleRelease`** (and after any change to `components/editor/cm-source.js`),
  or the APK will lack the editor assets and show a blank editor.
- For Play Store, build an **AAB** (`eas build` / `bundleRelease`) so per-device
  ABI splits keep the download small.
